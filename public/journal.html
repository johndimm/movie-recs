Movie Recs

Dev Journal — Sunday, April 12, 2026

The Idea

Starting prompt (yours): TikTok doesn't ask you what you like — it watches how you react. Apply the same idea to movies: an LLM picks a title, predicts your rating, you rate it 0–100, and the error is revealed. Over time the AI gets better. The goal is to minimize error.
Concept

Core mechanic

Each round: LLM selects a movie/TV title and makes a hidden prediction. User rates 0–100. Prediction is revealed alongside the error. A running-average error chart at the top tracks improvement over time. No title is ever shown twice.

Your Design Decisions

UX

Slider instead of number input

The original number input felt like filling out a form. You asked for a slider, which turned out to feel much more like a judgment — drag to where it feels right, hit Submit. The live large-number display next to the label replaced the need for any typed value.

UX

Remove the Next button — auto-advance after rating

After submitting a rating, the original design paused to show a result screen with a Next button. You pointed out the extra click was friction: show the result info below the chart instead, and immediately load the next movie. This made the loop feel like a card swipe rather than a form submission.

Feature

Movie details: poster, cast, plot

You asked for the main actors, a plot summary, and a poster or image on each card. Claude now returns cast and plot as structured JSON fields; the Serper image search API fetches a poster. The card became a proper movie entry rather than a bare title.

Feature

Rotten Tomatoes score

You wanted a professional rating to show alongside yours after the reveal. Claude returns the RT Tomatometer from its training knowledge (e.g. “91%”). A 🍅 / 💀 badge renders on the card and in the last-result panel, giving immediate external context for how your taste compares to critics.

Feature

Watchlist from “Haven’t seen it”

You noticed that the app was surfacing genuinely interesting films you hadn’t seen. Rather than just skipping them, you asked that they be saved. Now clicking “Haven’t seen it” adds the full card (poster, cast, plot, RT score, year) to a persistent watchlist page at /watchlist, with a count badge in the header.

Feature

Release year

A small but grounding detail — you asked to see when a film was made. The year now appears inline next to the type label (“Movie · 1994”) on both the main card and the watchlist.

Bugs Fixed

Fix

SVG NaN errors in the error chart

On first render the chart produced NaN for SVG attributes (y, height, path d). Root causes: dividing by zero when n−1 = 0 (single data point), and stale localStorage entries with no error field. Fixed by filtering invalid entries, centering the single-point case, and rendering a <circle> instead of a path for one data point.

Fix

“Haven’t seen it” showed the same movie again

Skipped titles were not being sent to the API, so the LLM could pick the same one again. Added a separate skipped list in localStorage, passed alongside the rating history, and merged both into the exclusion list in the prompt.

Fix

AI Predicted showed blank / Error showed NaN

The LLM returned predicted_rating (snake_case) but the TypeScript interface expected predictedRating (camelCase). A type assertion (as NextMovieResponse) silently accepted the wrong shape. Fixed by explicitly mapping the raw field to the camelCase key after parsing.

Chart Improvements

UX

Accuracy instead of error — up is good

You asked to flip the chart so that up means better performance, not worse. The Y axis now shows accuracy (100−error) rather than raw error. Grid reference lines at 75 (want-to-see penalty level) and 50 (not-interested penalty level) are preserved, and bar heights grow upward from the baseline.

Fix

Chart started at 100 on the first pick

The cumulative average was inflated by early lucky guesses and never recovered — the first pick happened to land well, pushing the average to ~100, and every subsequent mistake looked like a regression. Switched to a rolling window of the last 5 decisions (WINDOW = 5) so the line reflects recent performance rather than all-time history, making genuine improvement visible.
